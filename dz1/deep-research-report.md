# Универсальная доработка для 1С: ИИ-чат с возможностью выполнения действий

## 1. Коммерческое описание решения

**ИИ-чат для 1С** — это универсальная доработка, которая добавляет в систему современный, красивый и удобный чат-помощник. Пользователь может общаться с ИИ прямо внутри 1С: задавать вопросы, получать подсказки, анализировать данные, формировать отчеты и запускать разрешенные действия.

Решение ориентировано на пользователей, которым нужно быстрее находить информацию, понимать ошибки, работать с документами и получать помощь без долгого поиска по разделам, отчетам и инструкциям.

ИИ-чат работает в диалоговом режиме: пользователь пишет вопрос обычным языком, а система помогает ему выполнить задачу.

Примеры запросов:

```text
Покажи задолженность по контрагенту Альфа.

Почему не проводится этот документ?

Сформируй отчет по продажам за апрель.

Создай задачу Иванову проверить оплату по заказу.

Как оформить возврат товара от клиента?
```

Главная идея решения — сделать 1С более понятной, быстрой и удобной для пользователя.

---

## 2. Цель доработки

Цель доработки — встроить в 1С интеллектуального помощника, который:

* отвечает на вопросы пользователей;
* помогает работать с данными 1С;
* анализирует текущие документы;
* формирует отчеты по запросу пользователя;
* создает задачи после подтверждения;
* объясняет порядок работы в 1С;
* соблюдает права доступа и правила безопасности;
* сохраняет историю общения и журнал действий.

Документ составлен в формате концепции и технического задания без программного кода, в логике шаблона исследования/ТЗ: цель, контекст, основной вопрос, ограничения, требования к результату и дальнейшие рекомендации. 

---

## 3. Область применения

Доработка проектируется как **универсальное решение для 1С** с возможностью адаптации под разные конфигурации.

В документе решение рассматривается универсально, но с примерами для:

* **1С:Управление торговлей**;
* **1С:ERP**;
* **1С:Комплексная автоматизация**;
* **1С:Бухгалтерия**.

Решение не должно быть жестко привязано к одной конфигурации. Конкретный состав доступных объектов, отчетов и действий определяется при внедрении.

## 4. Основные возможности первой версии

В первую версию входят следующие возможности.

### 4.1. Общий ИИ-чат

В 1С появляется отдельная форма чата, где пользователь может общаться с ИИ.

Основные функции:

* ввод вопроса пользователем;
* получение ответа от ИИ;
* сохранение истории переписки;
* продолжение диалога в рамках одной сессии;
* отображение сообщений в современном и привлекательном интерфейсе;
* возможность открыть прошлые диалоги.

---

### 4.2. Контекстный ИИ-помощник

Помимо общего чата, пользователь может запускать ИИ из объектов 1С.

Например, пользователь находится в документе **“Заказ клиента”** и нажимает кнопку **“Спросить ИИ”**.

В этом случае ИИ получает контекст текущего объекта:

* тип объекта;
* ссылку на объект;
* основные реквизиты;
* табличные части;
* доступные пользователю действия;
* ограничения по правам.

Примеры запросов:

```text
Проверь этот заказ.

Почему документ не проводится?

Есть ли ошибки заполнения?

Что нужно исправить?

Создай задачу менеджеру по этому заказу.
```

---

### 4.3. Ответы на вопросы по данным 1С

ИИ может отвечать на вопросы пользователя по данным информационной базы.

Примеры:

```text
Какая задолженность у контрагента Альфа?

Какие продажи были за апрель?

Какие заказы сейчас в работе?

Какие товары заканчиваются на складе?

Покажи просроченную дебиторскую задолженность.
```

ИИ должен работать только с теми данными, которые доступны текущему пользователю по правам 1С и разрешены отдельными настройками безопасности ИИ.

---

### 4.4. Анализ текущего документа

ИИ может анализировать текущий документ и объяснять пользователю возможные проблемы.

Примеры:

```text
Проверь этот документ.

Почему он не проводится?

Что здесь заполнено неправильно?

Есть ли подозрительные данные?

Какие поля нужно заполнить?
```

ИИ может выявлять:

* незаполненные обязательные реквизиты;
* недостаток остатков;
* просроченную задолженность;
* некорректные даты;
* несоответствие сумм;
* возможные причины ошибки проведения.

---

### 4.5. Создание задач пользователям

ИИ может подготовить создание задачи пользователю, но не выполняет действие сразу.

Пример:

```text
Создай задачу Иванову проверить оплату по заказу №125 до завтра.
```

ИИ должен показать пользователю параметры будущего действия:

```text
Я создам задачу:

Исполнитель: Иванов И.И.
Срок: 09.05.2026
Текст: Проверить оплату по заказу клиента №125.

Подтвердить создание?
```

Действие выполняется только после нажатия кнопки **“Подтвердить”**.

---

### 4.6. Формирование отчетов

Пользователь может запросить отчет обычным языком.

Примеры:

```text
Покажи продажи по менеджерам за апрель.

Сформируй отчет по остаткам на сегодня.

Покажи дебиторскую задолженность по клиентам.

Сравни продажи за март и апрель.
```

ИИ должен:

* определить подходящий отчет;
* предложить параметры;
* показать пользователю, что будет сформировано;
* открыть отчет после подтверждения;
* при возможности показать краткую сводку в чате.

---

### 4.7. Объяснение работы в 1С

ИИ может работать как справочный помощник.

Примеры:

```text
Как оформить возврат товара от клиента?

Где посмотреть взаиморасчеты?

Как провести реализацию?

Почему не виден товар на складе?

Что делать, если документ не проводится?
```

ИИ должен давать понятные пошаговые инструкции с учетом используемой конфигурации и доступных пользователю разделов.

---

## 5. Обязательное подтверждение действий

Все действия, которые изменяют данные 1С, выполняются только после явного подтверждения пользователя.

ИИ не должен самостоятельно:

* создавать документы;
* изменять данные;
* удалять объекты;
* запускать массовые операции;
* отправлять сообщения;
* создавать задачи;
* менять настройки.

Перед выполнением действия ИИ обязан показать:

* что будет сделано;
* с каким объектом;
* какие параметры будут использованы;
* кто будет исполнителем;
* какой ожидается результат.

Пользователь должен иметь варианты:

```text
Подтвердить
Изменить
Отмена
```

Ответственным за подтвержденное действие считается пользователь, который его подтвердил.

## 6. Интерфейс решения

Интерфейс должен быть комбинированным.

### 6.1. Общий чат

Отдельная форма для общения с ИИ.

Назначение:

* общие вопросы;
* поиск информации;
* объяснение работы в 1С;
* формирование отчетов;
* просмотр истории диалогов.

### 6.2. Контекстный запуск

Кнопка **“Спросить ИИ”** или **“ИИ-помощник”** может размещаться в формах документов, справочников и отчетов.

Назначение:

* анализ текущего объекта;
* объяснение ошибок;
* действия по текущему документу;
* создание задач по объекту;
* быстрые подсказки пользователю.

## 7. Универсальное подключение ИИ-провайдеров

Решение должно поддерживать подключение разных ИИ-провайдеров через универсальный слой адаптеров.

Поддерживаемая концепция:

```text
ИИ-чат 1С
 ├─ интерфейс чата
 ├─ менеджер диалогов
 ├─ менеджер контекста 1С
 ├─ менеджер действий
 ├─ слой подтверждения
 ├─ контур безопасности
 └─ адаптеры ИИ-провайдеров
```

Возможные провайдеры:

* OpenAI / ChatGPT API;
* YandexGPT;
* GigaChat;
* локальная LLM-модель;
* другой REST API.

Бизнес-логика чата не должна зависеть от конкретного ИИ-сервиса.

Для каждого провайдера должен быть отдельный адаптер, отвечающий за:

* авторизацию;
* формат запроса;
* отправку сообщения;
* получение ответа;
* обработку ошибок;
* обработку лимитов;
* настройку модели;
* параметры безопасности.

---

## 8. Хранение истории и журнал действий

История общения с ИИ должна сохраняться постоянно.

Для каждого диалога фиксируются:

* пользователь;
* дата и время;
* текст запроса;
* ответ ИИ;
* контекст запуска;
* выбранный ИИ-провайдер;
* статус обработки;
* ошибки, если они были.

Отдельно ведется **журнал действий ИИ**.

В журнале фиксируются:

* какое действие предложил ИИ;
* какие параметры были показаны пользователю;
* подтвердил ли пользователь действие;
* отменил ли пользователь действие;
* было ли действие изменено;
* результат выполнения;
* ссылка на созданный или измененный объект;
* дата и время;
* пользователь, подтвердивший действие.

Пример записи:

```text
Пользователь: Иванов И.И.
Дата: 08.05.2026 10:35
Контекст: Заказ клиента №125
Запрос: Создай задачу менеджеру проверить оплату
Предложенное действие: Создание задачи
Статус: Подтверждено пользователем
Результат: Задача создана
```

---

## 9. Права доступа и безопасность

ИИ не должен становиться способом обхода прав 1С.

Доступ проверяется на двух уровнях:

```text
1. Права текущего пользователя 1С
2. Дополнительные правила безопасности ИИ
```

Если пользователь не имеет доступа к данным, ИИ также не должен иметь доступа к этим данным.

Дополнительно должны быть настройки:

* какие разделы доступны ИИ;
* какие документы доступны ИИ;
* какие справочники доступны ИИ;
* какие отчеты можно формировать через чат;
* какие действия разрешены;
* какие действия запрещены;
* какие данные нужно маскировать;
* какие роли могут использовать ИИ;
* какие пользователи могут создавать задачи через ИИ;
* какие пользователи могут просматривать журнал ИИ.

## 10. Контур безопасности и модерации ИИ-запросов

В решении должен быть отдельный контур безопасности.

Он выполняет проверки:

```text
Пользовательский запрос
   ↓
Проверка прав пользователя 1С
   ↓
Проверка правил безопасности ИИ
   ↓
Подготовка и фильтрация контекста
   ↓
Маскирование чувствительных данных
   ↓
Отправка запроса ИИ-провайдеру
   ↓
Проверка ответа ИИ
   ↓
Показ действия пользователю
   ↓
Подтверждение
   ↓
Выполнение действия
   ↓
Запись в журнал
```

Контур безопасности должен запрещать:

* запросы не по рабочему назначению;
* раскрытие технических деталей системы;
* доступ к данным без прав;
* передачу конфиденциальных данных без маскирования;
* выполнение действий без подтверждения;
* выполнение неразрешенных действий;
* массовые изменения;
* удаление данных;
* обход ролей и прав;
* инструкции, которые могут повредить информационную базу.

## 11. Уровни риска операций

Операции желательно разделить по уровню риска.

### Низкий риск

* справочный ответ;
* объяснение порядка работы;
* анализ без изменения данных;
* подсказка пользователю.

### Средний риск

* формирование отчета;
* анализ документа;
* подготовка задачи;
* поиск данных по разрешенным объектам.

### Высокий риск

* создание объектов;
* изменение данных;
* массовые операции;
* работа с персональными данными;
* работа с финансовыми данными;
* изменение настроек.

В MVP допускаются низкий и средний риск, а создание задач — только как контролируемое действие после подтверждения.

## 13. MVP

В MVP включается безопасная и полезная первая версия решения.

### Состав MVP

1. Красивый общий ИИ-чат в 1С.
2. История сообщений и диалогов.
3. Контекстный запуск из объектов 1С.
4. Ответы на вопросы по данным 1С.
5. Анализ текущего документа.
6. Формирование отчетов через чат.
7. Создание задач пользователям после подтверждения.
8. Объяснение порядка работы в 1С.
9. Универсальный слой ИИ-провайдеров.
10. Настройки доступа для ИИ.
11. Контур безопасности и модерации.
12. Журнал действий ИИ.

### Что не входит в MVP

В первую версию не включаются:

* автозаполнение полей документов;
* массовое изменение данных;
* удаление объектов;
* автоматическое выполнение действий без подтверждения;
* изменение настроек учета;
* критичные операции без дополнительного контроля;
* голосовой ввод;
* сложные аналитические рекомендации;
* RAG по корпоративной базе знаний.

## 14. Дальнейшее развитие

После MVP решение можно развивать поэтапно.

### 14.1. RAG по базе знаний компании

ИИ сможет отвечать на основе:

* инструкций;
* регламентов;
* методичек;
* внутренних правил;
* описаний бизнес-процессов;
* пользовательской документации.

### 14.2. Расширенные действия в 1С

Можно добавить:

* заполнение отдельных безопасных полей;
* подготовку документов;
* создание черновиков объектов;
* запуск регламентированных сценариев;
* расширенные действия после подтверждения.

### 14.3. Голосовой ввод

Пользователь сможет диктовать запрос голосом, а ИИ будет обрабатывать его как текстовую команду.

### 14.4. Персональные подсказки

ИИ сможет подсказывать пользователю:

* какие документы требуют внимания;
* какие задачи просрочены;
* какие заказы не обеспечены;
* где есть ошибки заполнения;
* какие операции нужно завершить.

### 14.5. Аналитические рекомендации

ИИ сможет выявлять:

* отклонения продаж;
* проблемных контрагентов;
* просроченную дебиторку;
* нехватку товаров;
* необычные изменения показателей;
* потенциальные ошибки учета.

---

## 15. Ограничения и риски

### 15.1. Ошибки ИИ

ИИ может неправильно понять запрос, выбрать не тот объект или предложить некорректное действие.

**Меры снижения риска:**

* обязательное подтверждение изменений;
* показ параметров перед выполнением;
* журналирование действий;
* запрет критичных операций в MVP.

---

### 15.2. Права доступа

ИИ не должен раскрывать данные, недоступные пользователю.

**Меры снижения риска:**

* проверка прав пользователя 1С;
* отдельные правила доступа ИИ;
* отказ при отсутствии прав.

---

### 15.3. Конфиденциальность данных

При использовании внешних ИИ-провайдеров часть данных может передаваться за пределы информационной базы.

**Меры снижения риска:**

* фильтрация контекста;
* маскирование чувствительных данных;
* настройка разрешенных объектов;
* возможность подключения локальной LLM-модели.

---

### 15.4. Стоимость ИИ API

Внешние ИИ-сервисы могут тарифицироваться по запросам, токенам или объему обработки.

**Меры снижения риска:**

* лимиты по пользователям;
* лимиты по ролям;
* лимиты по подразделениям;
* ограничения по типам запросов;
* мониторинг расходов.

---

### 15.5. Качество данных в 1С

Если данные в 1С неполные или ошибочные, ИИ может сделать неверный вывод.

**Меры снижения риска:**

* указание источника данных;
* указание периода анализа;
* предупреждение о неполных данных;
* сохранение ответственности пользователя за принятие решений.

---

### 15.6. Ответственность за действия

ИИ не должен считаться самостоятельным исполнителем операций.

**Меры снижения риска:**

* действие выполняется только после подтверждения;
* ответственным считается пользователь;
* все действия фиксируются в журнале.

## 17. Итоговая формулировка решения

**Универсальная доработка “ИИ-чат для 1С”** — это встроенный интеллектуальный помощник, который позволяет пользователям общаться с ИИ в режиме чата, получать ответы по данным 1С, анализировать документы, формировать отчеты, создавать задачи после подтверждения и получать инструкции по работе в системе.

Решение предусматривает красивый пользовательский интерфейс, контекстный запуск из объектов 1С, поддержку разных ИИ-провайдеров, постоянное хранение истории, журнал действий, контроль прав доступа, отдельный контур безопасности и обязательное подтверждение любых изменений данных.

MVP ориентирован на безопасные сценарии: ответы, анализ, отчеты, задачи и обучение пользователей. Дальнейшее развитие может включать RAG по базе знаний, расширенные действия в 1С, голосовой ввод, персональные подсказки и аналитические рекомендации.
