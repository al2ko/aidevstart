# KA-0001. Внешняя обработка для просмотра задач Bitrix24

## Бизнес-ситуация

Пользователь работает с задачами в Bitrix24, но часть операционной работы выполняет в 1С.
Сейчас для просмотра незакрытых задач приходится отдельно переходить в Bitrix24 и вручную сверять задачи по проектам.
Нужно упростить контроль текущей загрузки пользователя прямо из 1С: получать актуальные незакрытые задачи по токену Bitrix24 и показывать их в удобном сгруппированном виде.

## Что нужно сделать

1. Разработать внешнюю обработку 1С на базе заготовки `src/epf/ЗадачиИзБитрикс24`.
2. Реализовать подключение к Bitrix24 по токену и получение незакрытых задач пользователя.
3. Вывести найденные задачи в поле вывода обработки, сгруппировав их по проекту.

## Какие объекты исследовать / дорабатывать

- `ВнешняяОбработка.ЗадачиИзБитрикс24`
- Форма внешней обработки
- Модуль формы внешней обработки
- При необходимости: модуль объекта внешней обработки
- API Bitrix24 для получения задач пользователя; справку по подключению и методам получать через MCP `b24-dev-mcp`

## Ограничения

- Правки выполнять только во внешней обработке `src/epf/ЗадачиИзБитрикс24`.
- Не изменять объекты основной конфигурации и структуру метаданных основной базы.
- Токен Bitrix24 не хранить в коде обработки.
