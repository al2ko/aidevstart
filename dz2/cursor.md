# Анализ: «Недостаточно прав доступа» — АРМ рабочий по полиолу

## Контекст

При работе в обработке `Обработка.ай_АРМ_РабочийПоПолиолу`, форма «Производство», на последнем шаге при выполнении команды проведения куба возникает ошибка доступа к методу контекста `Получить` при обращении к константе.

## Стек из журнала регистрации

1. `ОбщийМодуль.ай_ОбщегоНазначения` — `Возврат Константы.ай_ОрганизацияДляДокументов.Получить();`
2. `Обработка.ай_АРМ_РабочийПоПолиолу.Форма.Производство` — присвоение организации новому документу через `ай_ОбщегоНазначения.ОрганизацияДляДокументов()`
3. Вызов из клиентской процедуры после `ПроизвелКубНаСервере()`

## Прямая причина

Чтение значения константы через `Константы.ай_ОрганизацияДляДокументов.Получить()` выполняется в серверном контексте от имени пользователя и требует права **Чтение (Read)** на объект метаданных **`Константа.ай_ОрганизацияДляДокументов`**. У пользователя с ролью АРМ это право отсутствует — платформа возвращает «Нарушение прав доступа».

## Участки кода (репозиторий kamapek)

- Заполнение организации: `cf\DataProcessors\ай_АРМ_РабочийПоПолиолу\Forms\Производство\Ext\Form\Module.bsl` — функция `ПроизвелКубНаСервере()`, присвоение `НовыйДокумент.Организация = ай_ОбщегоНазначения.ОрганизацияДляДокументов()` (заявка 100949).
- Реализация: `cf\CommonModules\ай_ОбщегоНазначения\Ext\Module.bsl` — экспортная функция `ОрганизацияДляДокументов()` вызывает `Константы.ай_ОрганизацияДляДокументов.Получить()`.

## Роли в конфигурации

- Роль **`ай_АРМРабочийПоПолиолу`** (`cf\Roles\ай_АРМРабочийПоПолиолу\Ext\Rights.xml`) содержит права на обработку, каталоги, регистры и т.д., но **не включает** объект `Constant.ай_ОрганизацияДляДокументов`.
- В ролях **`ай_ЧтениеИПросмотрНастройки`** и **`ай_ДобавлениеИзменениеНастройки`** перечислены многие константы с префиксом `ай_`, но **`ай_ОрганизацияДляДокументов` в списках нет** — константа добавлена позже и не была включена в эти роли.

## Проверка через MCP (инфобаза)

Инструмент **`get_access_rights`** для объекта `Константа.ай_ОрганизацияДляДокументов` показал, что явная настройка прав по ролям в базе сводится к минимальному набору (в т.ч. полные права); для узких ролей без записи на эту константу эффективный доступ на чтение отсутствует.

## Вывод

Ошибка не связана с логикой проведения документа как таковой, а вызвана **отсутствием у роли пользователя права чтения константы `ай_ОрганизацияДляДокументов`**.

## Рекомендуемое административное исправление

Назначить ролям пользователей АРМ (как минимум **`ай_АРМРабочийПоПолиолу`**) права **Чтение** и при необходимости **Просмотр** на **`Константа.ай_ОрганизацияДляДокументов`**, обновить состав прав в информационной базе. При необходимости добавить ту же константу в роли настроек (`ай_ЧтениеИПросмотрНастройки` / `ай_ДобавлениеИзменениеНастройки`) для согласованности с остальными константами.

Изменение прикладного кода для устранения именно этой ошибки доступа не требуется.
