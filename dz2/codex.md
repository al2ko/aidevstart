# Анализ ошибки при создании Заказа на производство из Заказа покупателя

## Симптом

При нажатии в `e1cib/data/Документ.ЗаказПокупателя` на кнопку `Создать на основании -> Заказ на производство` возникает ошибка:

```text
Ошибка при вызове метода контекста (ОткрытьФорму)
{ай_Доработки Документ.ЗаказПокупателя.Форма.ФормаДокумента.Форма(18)}:
ОткрытьФорму("ОбщаяФорма.ай_ФормаВводаДаты", Новый Структура("ДатыИзТЧ", МассивДат)

[ОшибкаВоВремяВыполненияВстроенногоЯзыка]
по причине:
Недостаточно прав для просмотра
[НарушениеПравДоступа]
```

## Найденное место ошибки

Ошибка возникает в расширении `ай_Доработки`, в модуле формы документа:

`cfe/ай_Доработки/Documents/ЗаказПокупателя/Forms/ФормаДокумента/Ext/Form/Module.bsl`

Строка 18:

```bsl
ОткрытьФорму("ОбщаяФорма.ай_ФормаВводаДаты", Новый Структура("ДатыИзТЧ", МассивДат),
        ЭтотОбъект,,,,ОписаниеОповещения, РежимОткрытияОкнаФормы.БлокироватьОкноВладельца);
```

То есть ошибка возникает до фактического создания документа `ЗаказНаПроизводство`: сначала расширение открывает служебную общую форму выбора даты.

## Проверенная форма

Общая форма существует в расширении:

`cfe/ай_Доработки/CommonForms/ай_ФормаВводаДаты.xml`

Имя объекта метаданных:

```xml
<Name>ай_ФормаВводаДаты</Name>
```

Форма расположена здесь:

`cfe/ай_Доработки/CommonForms/ай_ФормаВводаДаты/Ext/Form.xml`

По структуре формы видно, что у нее нет основного реквизита прикладного объекта. Есть только реквизит формы `Дата` типа `xs:dateTime` и реквизит `ПараметрыВыполнения`. В модуле формы выполняется только заполнение списка выбора дат и закрытие формы с результатом.

## Проверка ролей

В расширении есть собственная роль:

`cfe/ай_Доработки/Roles/ай_ОсновнаяРоль.xml`

Но в выгрузке расширения не найден файл прав для этой роли:

```text
cfe/ай_Доработки/Roles/ай_ОсновнаяРоль/Ext/Rights.xml
```

Также поиском по ролям не найдено право на объект:

```text
CommonForm.ай_ФормаВводаДаты
```

В основной конфигурации есть примеры выдачи прав на другие общие формы `ай_`, например:

```text
cf/Roles/.../Ext/Rights.xml: CommonForm.ай_РезультатыПоискаПоШтрихкоду
```

Но для `CommonForm.ай_ФормаВводаДаты` право не найдено.

## Вывод

Вероятная причина ошибки: пользователю не выдано право `Просмотр` (`View`) на общую форму расширения:

```text
ОбщаяФорма.ай_ФормаВводаДаты
CommonForm.ай_ФормаВводаДаты
```

Так как это общая форма без основного реквизита объекта, для ее открытия требуется право просмотра именно на саму общую форму. Ошибка `Недостаточно прав для просмотра` на вызове `ОткрытьФорму` соответствует отсутствию этого права.

Дополнительный фактор: расширение содержит собственную роль `ай_ОсновнаяРоль`. По правилам работы расширений, если в расширении есть собственные роли, доступ к объектам расширения должен быть явно предоставлен через роли расширения или через доработанные роли основной конфигурации.

## Что проверить в базе

1. Назначена ли пользователю роль расширения `ай_ОсновнаяРоль`.
2. Есть ли в этой роли право `Просмотр` на `ОбщаяФорма.ай_ФормаВводаДаты`.
3. Если доступ должен быть не через отдельную роль расширения, проверить, не нужно ли заимствовать/доработать существующую пользовательскую роль и добавить туда право `View` на `CommonForm.ай_ФормаВводаДаты`.
4. После изменения прав обновить расширение в базе и переподключить пользователя, чтобы права перечитались.

## Использованные MCP-проверки

- `1c-template-search-mcp.recall` — проектная память по похожей ошибке пустая.
- `1c-template-search-mcp.templatesearch` — проверены типовые паттерны открытия форм.
- `1c-code-checker-naparnik.onec_help` — проверка по платформенной документации по `ОткрытьФорму` и режимам открытия.
- `1c-code-checker-naparnik.ask_1c_ai` — подтверждено, что для пустой общей управляемой формы требуется право `Просмотр` на саму общую форму.
- `1c-code-checker-naparnik.its_help` + `fetch_its`:
  - `its-metod8dev-5929-hdoc` — права доступа к формам в 1С:Предприятии, общие формы являются исключением и права на них настраиваются отдельно.
  - `its-pubextensions-66-hdoc` — роли в расширениях: при наличии собственных ролей доступ к объектам расширения должен предоставляться через роли.
